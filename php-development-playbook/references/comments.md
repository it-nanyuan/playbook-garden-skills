# PHP Comments Standard

## Core Principles

- 注释优先解释职责、约束、副作用和业务意图
- 不用注释重复代码表面动作
- 对公共能力、复杂流程和风险链路优先补足说明

## Class Comments

- Service：写清业务边界、适用场景和是否涉及事务、缓存、队列或外部调用
- Repository：写清数据来源、查询特点和是否封装复杂查询
- Job / Listener：写清触发来源、重试策略、幂等要求和失败影响
- Controller：只在职责不明显或存在聚合入口时补说明

## Method Comments

- 说明输入、输出、副作用、异常行为、幂等要求和前置条件
- 复杂聚合查询、支付链路、状态流转、回调处理、批处理、导入导出优先写意图说明

## Property Comments

- 状态字段、金额字段、时间单位、枚举值和配置项在语义不明显时补说明
- 对数组结构、混合对象和第三方返回字段做必要标注

## Special Cases

- Webhook / 支付回调：注明验签、幂等、防重放和状态更新顺序
- 缓存更新：注明 key 规则、过期时间、删除策略和一致性要求
- 队列任务：注明重试、超时、失败补偿和告警方式
