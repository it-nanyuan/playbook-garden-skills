# PHP Error Handling Standard

## Error Categories

- 参数错误
- 权限错误
- 业务错误
- 系统错误
- 第三方依赖错误

## Rules

- 对外错误响应统一，不暴露堆栈和敏感实现细节
- 内部日志保留 traceId、关键主键、请求来源和异常上下文
- 全局异常处理统一收口
- 第三方调用失败、队列失败、支付失败要区分可重试与不可重试

## Common Scenarios

- 参数校验失败：直接返回统一格式错误，不进入业务逻辑
- 支付回调异常：先记日志，再按协议决定是否返回重试信号
- 后台批量任务异常：保留失败明细、补偿入口和告警信息
