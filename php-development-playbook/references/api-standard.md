# PHP API Standard

## Unified Response

- 统一成功 / 失败返回体结构
- 统一错误码、错误消息和追踪信息字段
- 统一分页对象命名与字段结构

## Layering

- Controller 只处理协议层、参数接收、权限门面和响应转换
- Service 返回业务结果，不把 HTTP 细节扩散到业务层
- Model 不直接裸返回给前端

## Validation

- 参数校验优先使用框架原生能力
- 校验规则集中维护，不在每个方法里零散拼接

## API Design

- 列表、详情、创建、更新、删除、导出接口风格一致
- 导入导出、支付回调、Webhook、批量操作明确幂等与权限要求
- 文件上传、下载、导出接口要额外关注安全边界和资源消耗
