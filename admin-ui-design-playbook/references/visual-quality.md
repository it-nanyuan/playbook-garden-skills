# Visual Quality

## Core Standard

目标是“专业、克制、可信、可商用”，不是“热闹、显眼、像概念图”。

## Typography

- 页面标题克制，不要满屏都是超大标题
- 一个页面通常只需要一个主标题
- 副标题和辅助说明尽量短
- 用字体层级解决信息关系，不靠堆字数

## Cards

- 卡片只在真正需要分组时使用
- 不要把每一小块信息都包成卡片
- 同一页面卡片样式数量要少
- 卡片内优先放关键指标、关键动作、清晰摘要

## Text

- 后台界面避免长段解释文字
- 提示文案只在能减少误操作时出现
- 优先让结构自己说话，而不是靠说明文字补救

## Color And Emphasis

- 颜色优先服务状态和层级，不是装饰
- 一个页面的主强调色要收敛
- 风险、警告、成功、处理中状态要稳定一致

## Spacing

- 间距要形成系统，不要一块紧一块松
- 页面块级节奏、表单控件间距、表格上下关系要统一
- 留白是为了建立秩序，不是为了显得“高级”

## Anti-Patterns

- 大标题配大段说明
- 卡片墙
- 背景、渐变、阴影全部抢内容
- 操作按钮分布零散
- 图标很多但语义很弱
- 每个模块都想当视觉中心
