# Admin UI Design Playbook for GitHub Copilot

Apply these rules when asked to design or implement admin UI, dashboards, or business web interfaces.

## Workflow

- Clarify users, tasks, competitors, style direction, and information density first.
- Do not jump directly to a generic mockup.
- Prefer layouts that feel commercially credible and operationally useful.

## Quality Standard

- Avoid oversized titles, long instructional text, and card walls.
- Use layout, grouping, and hierarchy to organize complexity.
- For admin pages, prioritize navigation clarity, filters, tables, status visibility, and operational efficiency.
- If implementing code, preserve the same design intent instead of reducing the page to plain scaffolding.
