# Tools

## CI/CD

- `Jenkins`
- `GitHub Actions`
- `GitLab CI`

## Container And Runtime

- `Docker`
- `Kubernetes`
- `Helm`
- `kubectl`

## Traffic And Delivery

- `Nginx`
- 云负载均衡
- 网关或入口控制器

## Registry And Artifacts

- `Harbor`
- 云镜像仓库
- 制品仓库

## Observability

- `Prometheus`
- `Grafana`
- `ELK` / `EFK`
- `SkyWalking`
- `Jaeger`
- `Alertmanager`

## Automation And Infrastructure

- `Terraform`
- `Ansible`

## Rule

- 工具链选择优先跟随现有基础设施
- 能统一的就统一，不要同类型工具混太多套
- 发布、监控、日志、回滚和配置至少要形成一条完整闭环
