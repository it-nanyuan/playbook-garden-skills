# Comments

## Goal

注释服务于协作、排障和后续维护，不是为了把代码逐句翻译成人话。

## Required

- 类、结构体、枚举、协议：写职责说明
- 公共方法：写作用、关键参数、返回值和副作用
- 异步方法：写线程、回调、取消或状态影响
- 复杂属性：只在业务含义不明显时补注释

## Avoid

- `// set value`
- `// update ui`
- `// request api`

## Good Bias

- 注释写“为什么这样做”
- 复杂条件分支、兼容逻辑、状态机、性能取舍附近补短注释
- 对外可复用 API 的注释优先级高于页面内部私有小函数
