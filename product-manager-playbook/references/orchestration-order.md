# Orchestration Order

## Default Order

1. 澄清需求
2. 如有必要先做调研、竞品、用户问题提炼
3. 确认范围和不做项
4. 定义验收标准
5. 判断是否先设计
6. 拆前端 / 移动端 / 后端 / 数据 / 运维任务
7. 进入测试拆解、回归验证和上线风险判断
8. 开发阶段受注释规范约束
9. 交付阶段受提交规范约束
10. 上线前后输出验收结论、总结和汇报

## Typical Examples

- 后台新功能
  - `product-manager-playbook`
  - `market-research-playbook`（如需）
  - `admin-ui-design-playbook`
  - `frontend-development-playbook`
  - 对应后端 skill
  - `dba-playbook`
  - `testing-playbook`
  - `devops-playbook`
  - `git-commit-playbook`

- App 功能
  - `product-manager-playbook`
  - `market-research-playbook`（如需）
  - `admin-ui-design-playbook`
  - 对应移动端 skill
  - 对应后端 skill
  - `testing-playbook`
  - `git-commit-playbook`

- 需求前置调研
  - `market-research-playbook`
  - `product-manager-playbook`

- 交付验收
  - `product-manager-playbook`
  - `testing-playbook`
