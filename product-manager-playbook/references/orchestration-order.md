# Orchestration Order

## Default Order

1. 澄清需求
2. 确认范围和不做项
3. 定义验收标准
4. 判断是否先设计
5. 拆前端 / 移动端 / 后端 / 数据 / 运维任务
6. 开发阶段受注释规范约束
7. 交付阶段受提交规范约束

## Typical Examples

- 后台新功能
  - `product-manager-playbook`
  - `admin-ui-design-playbook`
  - `frontend-development-playbook`
  - 对应后端 skill
  - `dba-playbook`
  - `devops-playbook`
  - `git-commit-playbook`

- App 功能
  - `product-manager-playbook`
  - `admin-ui-design-playbook`
  - 对应移动端 skill
  - 对应后端 skill
  - `git-commit-playbook`
