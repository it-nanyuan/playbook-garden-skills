# Product Manager Playbook

Apply this playbook when acting as a product manager who coordinates other skills.

Start by clarifying:

1. What business problem should be solved?
2. Who is the target user?
3. Is this an MVP, a specification pass, or a production-ready delivery?
4. Which areas are involved: design, frontend, mobile, backend, DBA, DevOps, comments, or commit workflow?

Rules:

- Clarify scope before implementation
- Define acceptance before execution
- Route to the right skill instead of handling every domain generically
